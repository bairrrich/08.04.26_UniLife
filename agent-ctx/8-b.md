### Task 2: Enhanced Notification Panel with Real Notifications

**Changes to `notifications-panel.tsx`:**

1. **Added `Target` icon** to imports and `ICON_MAP`

2. **Added helper functions**:
   - `pluralize(n, one, few, many)` — Russian pluralization (1 привычка, 2 привычки, 5 привычек)
   - `computeStreak(dates)` — Computes consecutive day streak from an array of dates, checking from today/yesterday backwards up to 365 days

3. **Contextual notification generation** (`fetchContextualNotifications`):
   - **Habit Reminders**: Fetches `/api/habits`, counts uncompleted habits (`!todayCompleted`), generates reminder with Russian pluralization

   - **Diary Streak**: Fetches `/api/diary`, computes consecutive day streak, shows "Серия дневника: X дней 🔥" if >= 3
   - **Workout Streak**: Fetches `/api/workout`, computes consecutive day streak, shows "Серия тренировок: X дней 💪" if >= 3
   - All fetches use `.catch(() => null)` for resilience — contextual notifications are optional enhancements

4. **Modified `fetchNotifications` callback**:
   - Now fetches both API notifications (`/api/notifications`) and contextual notifications in parallel via `Promise.all`
   - Merges results: contextual notifications first, then API notifications
   - Deduplication by `icon+module` key to avoid duplicate notifications for the same concern
   - Contextual notifications take priority over API ones

5. **Added `AllReadState` component**:
   - Green checkmark icon in a circle with gradient glow
   - "Всё прочитано" heading
   - "Новых уведомлений нет. Вы в курсе всех событий!" description
   - Same layout structure as `EmptyState` for visual consistency

6. **Modified rendering conditions** (4-branch ternary):
   - `loading` → Skeleton list
   - `!data || data.notifications.length === 0` → EmptyState (no notifications exist)
   - `unreadCount === 0` → AllReadState (all notifications read)
   - else → Notification list (has unread notifications)
